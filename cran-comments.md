## Resubmission
This is a resubmission of 3.1.0 addressing the issues raised by the CRAN
incoming pre-tests on 2026-07-16:

* Fixed the LaTeX/PDF-manual build failure. Non-ASCII box-drawing (U+2500)
  and em-dash characters in the examples of `man/remtribute.Rd` were replaced
  with ASCII. This resolves the "Unicode character not set up for use with
  LaTeX" errors and the associated PDF-manual WARNING/ERROR (and the leftover
  `remstimate-manual.tex` in the check directory, which was a side effect of
  the failed PDF build).
* Added a `\usage` section to `bic_table.Rd` (the S3 generic is exported via
  `@name`, so no usage was auto-generated by roxygen2).
* Converted all `\url{https://doi.org/...}` references to `\doi{}` in the Rd
  files.
* Changed the arXiv reference in DESCRIPTION from `<arXiv:2602.21000>` to
  `<doi:10.48550/arXiv.2602.21000>`, and corrected a typo ("extentions").

The remaining "possibly misspelled words" in DESCRIPTION are author surnames
(Juozaitiene, Karimova, Lakdawala, Leenders, Meijerink, Tibshirani) and the
standard abbreviations "et al." — these are correct as written.

## Reverse dependency (remverse)
remstimate 3.1.0 introduces changes that are not fully backward compatible
with the current CRAN version of the strong reverse dependency remverse.
remverse is maintained by us, and an updated version restoring full
compatibility with remstimate 3.1.0 is ready and will be submitted to CRAN
immediately after remstimate 3.1.0 is accepted. The reverse-dependency check
failure is therefore temporary and will be resolved by that follow-up
submission.
